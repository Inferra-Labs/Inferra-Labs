## 📏 Governance & Strict Rules

Inferra Labs operates with a strict internal code. This is not a college group or a trial-run community. This is a **mission-driven lab**, and each contributor is expected to uphold the following principles:

### 🔒 Core Rules

1. **Commit Discipline:**  
   - No direct commits to `main` or `production` branches.  
   - All changes must go through proper PR and review process.  
   - Semantic commit messages are mandatory.

2. **Communication Protocol:**  
   - **English is mandatory** in all documentation, meetings, GitHub comments, and discussions.  
   - Informal or unstructured communication is discouraged.

3. **Respect the Hierarchy:**  
   - Department Leads have complete technical and operational authority over their domain repositories.  
   - Respective Leads approve, reject, or assign tasks, and their word is final.

4. **No Passive Participation:**  
   - Inactivity > 3 days without prior Info will lead to automatic archival from org (unless approved by Team Lead).  
   - All members must either **contribute code, design, docs, or internal systems** regularly.

5. **Zero Excuse Culture:**  
   - Missed deadlines are not tolerated unless pre-approved.  
   - Punctuality and accountability are non-negotiable.

6. **Confidentiality and Integrity:**  
   - Internal documents, strategy decks, and pre-release code are not to be shared externally.  
   - Violation = permanent removal and report.

7. **Respect Code Quality:**  
   - No spaghetti code. No shortcuts.  
   - All contributions must follow project-level architecture & clean coding standards.
